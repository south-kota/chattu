The mounted disk image was inspected directly. Its application reports the expected name, bundle identifier, version, and `arm64` executable. It has an ad-hoc signature because this Mac has no Apple Developer signing identity configured.

The development service returns HTTP 200 with the title `Chattu (Alpha)`, and the Electron development process launches against the isolated Chattu home. The Codex in-app browser runs in an isolated environment that cannot access the Mac's loopback port, and macOS screen capture permission is unavailable to Codex, so no browser screenshot is included in this baseline.

An initial development launch reached `main window created` about 5.0 seconds after desktop runtime logging began, including first-run database migrations. An idle process-family sample used about 256 MiB resident memory and 0.4% aggregate CPU. These are development-build reference measurements, not production performance claims.

## Installation cleanup

The previously installed `/Applications/T3 Code (Alpha).app` bundle was quit and moved to Trash. Its 40 MB application-support directory at `~/Library/Application Support/t3code` was preserved. Chattu uses separate application and home directories so development does not mutate that retained data.
